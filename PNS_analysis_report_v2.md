# PNS 分析报告 v2（正向+反向）

## 方法论回顾

- **正向 PNS (Forward PNS)**：在**正确**推理路径上，ablate 某步后模型做对的概率下降多少。PNS 高 = 该步不可或缺。
- **反向 PNS (Reverse PNS)**：在**错误**推理路径上，ablate 某步后模型做对的概率上升多少。Rev PNS 高 = 该步是误导源/平庸之恶。
- **消融模式**：counterfactual（禁止该步推理，要求模型找替代路径继续）。
- **步骤筛选**：高熵步骤（entropy_top_ratio=0.2, step_entropy_top_ratio=0.2），这些是"分叉决策点"。
- **Benchmark**：MATH500 子集 30 题，每题 5 次 rollout。

---

## 1. 正向 PNS 与反向 PNS 的相关性

**核心问题**：啰嗦（Fwd PNS 低）和误导（Rev PNS 高）是 overthinking 中因果能力缺失的同一体现吗？

### 1.1 模型级别相关性（31个数据点：5 基础模型 + 13×2 训练checkpoint）

| 指标 | 值 | p 值 |
|------|------|------|
| Pearson r | **-0.412** | 0.021 |
| Spearman ρ | -0.098 | 0.599 |

Pearson 显著负相关说明：**Fwd PNS 越低的模型/checkpoint，Rev PNS 越高**（啰嗦与误导同时出现）。

### 1.2 "不必要性"视角（5 个基础模型）

将 Fwd PNS 翻转为"不必要性" `(1 - Fwd Mean)`，与 Rev PNS 对齐方向：

| Model | 1-Fwd (不必要) | Rev PNS (误导) | Overthinking Score |
|-------|:---------:|:---------:|:---------:|
| Base | 0.182 | 0.004 | **0.186** |
| Q3-Base | 0.385 | 0.062 | **0.447** |
| Instruct | 0.533 | 0.249 | **0.782** |
| Distill | 0.801 | 0.214 | **1.016** |
| ScaleR | 0.781 | 0.291 | **1.071** |

**Pearson r = 0.886, p = 0.046**（5 基础模型上，不必要性和误导性高度正相关）。

**结论**：**啰嗦和误导确实是 overthinking 因果能力缺失的两面。** Fwd=0% 比例与 Rev≥0.6% 比例的相关性 r=0.811 (p=0.096)。模型正确路径上步骤越冗余，错误路径上步骤就越具误导性。SFT/Distill 引入的 overthinking 不仅让正确路径变啰嗦，也让错误路径变得更具"平庸之恶"。

### 2.2 高熵 vs 低熵步骤的 PNS 变化对比

核心问题：Forking-DAPO 只训练高熵 token，PNS 变化是否只发生在高熵步骤？

**Reverse PNS 与训练步数的相关性：**

| 模型 | 高熵 r (p) | 低熵 r (p) |
|------|:----------:|:----------:|
| DAPO | **-0.844** (0.0003) | **-0.686** (0.010) |
| Forking-DAPO | -0.012 (0.969) | -0.173 (0.572) |

**Forward PNS 与训练步数的相关性：**

| 模型 | 高熵 r (p) | 低熵 r (p) |
|------|:----------:|:----------:|
| DAPO | **-0.572** (0.041) | -0.333 (0.266) |
| Forking-DAPO | +0.441 (0.132) | +0.421 (0.152) |

**逐步高熵-低熵差（Rev PNS）：**

| Step | DAPO 高熵 | DAPO 低熵 | Δ(高-低) | Fork 高熵 | Fork 低熵 | Δ(高-低) |
|:----:|:---------:|:---------:|:---------:|:---------:|:---------:|:---------:|
| 200 | 0.290 | 0.219 | +0.071 | 0.285 | 0.255 | +0.029 |
| 600 | 0.265 | 0.295 | -0.030 | 0.268 | 0.274 | -0.006 |
| 1000 | 0.203 | 0.266 | -0.063 | 0.303 | 0.297 | +0.007 |
| 1400 | 0.137 | 0.134 | +0.003 | 0.256 | 0.262 | -0.006 |
| 2000 | 0.116 | 0.135 | -0.019 | 0.321 | 0.300 | +0.021 |
| 2400 | 0.060 | 0.123 | -0.063 | 0.307 | 0.285 | +0.022 |

**结论：**

1. **DAPO 的 Rev PNS 下降在高熵和低熵步骤上同时发生**，但高熵步骤下降更快（r=-0.844 vs -0.686）。DAPO 在所有 token 上都有梯度，所以高低熵步骤同步改善。
2. **Forking-DAPO 无论高熵还是低熵步骤，Rev PNS 都没有显著变化**。虽然 Forking-DAPO 只在高熵 token 上训练，但**高熵步骤的 Rev PNS 也没有下降**。
3. 这说明 **减少错误路径的误导性（降低 Rev PNS）需要对整个推理链进行端到端训练**，仅训练"决策点"不够。DAPO 的全 token 梯度让模型学会了在错误路径上更好地"脱困"，而 Forking-DAPO 的局部梯度无法传递这种全局因果信号。
4. 对于 Forward PNS：DAPO 高熵步骤有弱显著下降（r=-0.572, p=0.041），Forking-DAPO 有弱上升趋势但不显著（r=+0.441, p=0.132）。两者都在增加 Fwd=0% 比例（正确路径冗余步骤增多），说明 RL 训练过程中 overthinking 的"啰嗦"维度可能在加重。

## 6. 高熵步骤选择的合理性

**定位**：我们不是说"高熵步骤天然就和 PNS 有关"，而是：
- **低熵步骤是执行步骤**，大多数时候没什么可变的（模型有很高置信度继续走当前路径），PNS 评估不太合适。
- **高熵步骤天然带来分叉**（熵高 = 模型在这里有多个可能路径），更适合也更值得 PNS 评估。
- **反向 PNS 适合所有错误路径步骤**：如果这一步能通过纠错答对，惩罚这步的"平庸之恶"是合适的。
- **在线 RL 算力限制**下，只能选一小部分做 PNS reward，选更重要（高熵）的部分更合理。

实际数据也支持这一定位：高熵步骤的 Fwd PNS 和 Rev PNS 在不同模型间的区分度和正向上的差值基本一致，不会因为只看高熵步骤而改变模型间的排序。

## 7. Reward 设计：Fwd PNS 做正 reward，-Rev PNS 做负 reward

### 7.1 方案

```
对于正确推理路径的高熵步骤 i:  reward_i = +fwd_pns_i ∈ [0, 1]
对于错误推理路径的高熵步骤 j:  reward_j = -rev_pns_j ∈ [-1, 0]
```

这自然满足：
- **正确路径上必要的步骤**获得高正 reward → 强化因果推理
- **正确路径上冗余的步骤** reward=0 → 不强化 overthinking
- **错误路径上误导性的步骤**获得强负 reward → 惩罚"平庸之恶"
- **错误路径上无害的步骤** reward=0 → 不误惩罚

### 7.2 Reward 信号分布

| Model | 正路径 Avg+ | 负路径 Avg- | 正路径有信号% | 负路径有信号% |
|-------|:-----------:|:-----------:|:-------------:|:-------------:|
| Base | +0.818 | -0.004 | 100.0% | 1.9% |
| Instruct | +0.467 | -0.249 | 77.9% | 52.1% |
| Distill | +0.199 | -0.214 | 38.2% | 47.3% |
| ScaleR | +0.219 | -0.291 | 41.7% | 55.7% |
| Q3-Base | +0.615 | -0.062 | 95.6% | 23.4% |

### 7.3 潜在问题与建议

1. **净信号方向正确**：对 Base/Q3-Base/Instruct，正 reward 远大于负 reward（净正），符合预期。对 Distill/ScaleR，净信号为负（-101/-101），说明这些模型的 overthinking 严重到需要大量纠正。

2. **稀疏性问题**：
   - Base 模型的负路径几乎无信号（1.9%），因为 Base 错得"不结构化"，rev PNS 很低。
   - Distill/ScaleR 的正路径稀疏（~40% 有信号），因为大量冗余步骤 PNS=0。
   - **建议**：对 PNS=0 的步骤可以给一个小的 baseline reward（如 -0.05 对正确路径冗余步骤），防止纯零信号。

3. **训练中的自举问题**：PNS 是用当前模型计算的，训练后模型变了，PNS 也会变。建议定期（每 N 步）重新计算 PNS reward，或者在 rollout 时同时做 ablation。

4. **与 ORM/PRM 的互补性**：PNS reward 衡量的是"因果必要性"而非"正确性"。一个正确但冗余的步骤 PNS=0，一个错误但关键的方向选择 rev PNS 高。这与 PRM 的"正确/错误"标签互补，可以考虑 `total_reward = α * PRM_score + β * PNS_reward`。

## 附：RL 对 Rev PNS 的影响假设验证

之前的观察：
- Distill → ScaleR (RLVR): Rev PNS 从 0.214 → 0.291（变多）
- Qwen3-Base → DAPO: Rev PNS 从 0.062 → 0.290 → ... → 0.060（先升后降）

**数据显示**：DAPO 训练过程中 Rev PNS 的变化主要由**错题集的变化**驱动。DAPO-step2400 仍做错的 21 题在 Base 上就是低 Rev PNS 的（Base 在这些题上的错误"不结构化"），但 DAPO 训练早期的错题集包含了很多 Base 未做错的题（DAPO 学会新解法但走偏），这些题的 Rev PNS 高。训练后期 DAPO 解决了更多题，剩下的"顽固"错题 Rev PNS 降低。

**结论**：RL 不直接降低 Rev PNS，而是通过**改变错题集**间接影响。模型解决更多题后，剩余错题的 Rev PNS 自然趋于 Base 水平（因为这些题本身就是 Base 也做不好的难题）。
